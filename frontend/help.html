v1.0
What this app does

ForzaTek is a data collection, annotation, training, and deployment platform for building a custom perception model that lets an AI drive in Forza Horizon. You collect frames, label a few hundred, train a model, let it label the rest, iterate, then eventually use the trained model to drive the car.

This app is the foundation. Perception and active-learning are fully implemented (stages A-D). Live screen capture, driving policy, and virtual controller output (stages E-G) will be wired up in the next phase.

Workflow · seven stages
A
Collect frames · record or ingest
Record live Forza via UDP + screen capture, or paste YouTube URLs, or drop .mp4 files. Target 2,000–5,000 raw frames across different biomes, weather, and time of day.
B
Label ~500 frames by hand
Paint road / curb / wall / offroad masks. Draw bounding boxes on vehicles and signs. Use pre-label to seed each frame with a rough guess, then correct. Budget 8–12 hours.
C
Train round 1
20 minutes on an RTX 4080. EfficientNet-B1 backbone with segmentation + detection heads. Expect mIoU ~0.64 on round 1. Not great yet, but the foundation is there.
D
Active learning · rounds 2, 3, 4
Model labels 500 more frames automatically. Queues the 50 most uncertain for your review. You accept or fix. Retrain. Repeat until mIoU plateaus (~0.85+).
E
Live perception · next phase
mss or DXGI screen capture, runs the trained model at 60 FPS. Outputs segmentation mask + detection boxes in real time for the driving policy to consume.
F
Driving policy · next phase
Start with rule-based: PID controller against the road centerline + simple throttle/brake logic. An RL agent (PPO/SAC) is a later, much bigger project.
G
Controller output · next phase
vgamepad library creates a virtual Xbox controller. Forza sees it as real hardware. Steering / throttle / brake values come from the driving policy, sent at 60 Hz.
Quick start
  1. Enable Data Out in Forza HUD settings, port 5300, format dash
  2. Run python -m backend.main from the project root
  3. Open http://localhost:8000 in your browser
  4. Record or ingest 2,000+ frames
  5. Label 500 frames from the Label page
  6. Run python -m scripts.run_pipeline --round 1
  7. Review the 50 uncertain frames on the Label page
  8. Run round 2, round 3, etc. until mIoU plateaus
Troubleshooting
Backend shows OFFLINE
Make sure python -m backend.main is still running in a terminal. The sidebar dot polls /api/system/health every 3 seconds.
No telemetry data
Check Forza HUD settings. Data Out must be ON, port 5300, format dash. Windows Firewall sometimes blocks the UDP packet — allow Python through.
Training crashes with CUDA out of memory
Drop batch size to 4 or 2 on the Train page config, or switch to a smaller backbone (edit training/model.py).
YouTube download fails
yt-dlp breaks often as YouTube changes. Update it: pip install -U yt-dlp
Pre-label button does nothing
The pre-labeler needs torch + torchvision + ultralytics installed. First call is slow (downloads the pretrained weights). Check the backend terminal for errors.
